# /회의시작

CEO가 "회의 시작해" 또는 `/회의시작 [주제]` 라고 하면 강팀장이 아래 프로토콜을 *자동 수행*한다.

주제(`$ARGUMENTS`)가 비어 있으면 강팀장이 *프로젝트를 스캔*해서 가장 시급한 주제 1개를 직접 정한다.

---

## 자동 회의 프로토콜 (강팀장 진행)

### 1) 회의 폴더 생성

```bash
bash ~/.claude/bin/start-meeting.sh "$ARGUMENTS"
# 인자 없으면 강팀장이 주제를 한 줄로 정한 뒤 다시 실행
```

### 2) 프로젝트 스캔 → 안건 생성

강팀장은 *동시에* 다음을 수집한다 (병렬 Bash·Read 호출):
- `git log -20 --oneline` — 최근 변경
- `git diff HEAD~5 --stat` — 변경 범위
- `ls .ai-team/meetings/ 2>/dev/null | tail -5` — 최근 회의록
- `find . -name "*.md" -path "*/team-memory/*"` — 전역 브레인에 누적된 학습
- `ls src/ app/ pages/ 2>/dev/null` — 코드 구조
- `git log --since="2 weeks ago" --pretty=format:"%s"` — 최근 작업 흐름

위 정보로 *4개 영역*에 대해 *각 1~2개씩* 안건을 뽑는다:
- 🧪 **UX 개선** — 사용자가 막힐 만한 흐름·이탈 지점
- 🎨 **UI 개선** — Crowny Class 토큰 적용 누락·디자인 부채
- 💡 **새 아이디어 (수익)** — 지속 사용 + 심플 수익으로 연결 가능한 한 가지
- 🛡️ **보안·리스크** — 인증·결제·개인정보·API 키 노출
- 🔍 **품질** — 최근 회귀·미테스트 영역 (선택)

### 3) 안건 작성 — `meeting.html` 첫 턴

강팀장 턴 블록을 `<!-- TURNS_START -->` 바로 아래에 Edit으로 append:

```html
<div class="turn pm">
  <div class="who"><span class="emoji">🎯</span> 강팀장</div>
  <div class="bubble">
    <p><strong>안건</strong></p>
    <ol>
      <li>[영역] 한 줄 안건</li>
      ...
    </ol>
    <p><strong>참석</strong>: 강팀장·강디·강개발·강체크·아뱅</p>
    <p><strong>목표</strong>: 지속 사용 + 심플 수익. 한 회의 = 한 결정.</p>
  </div>
</div>
```

### 4) 각 멤버 발언 — Agent 도구로 *병렬* 호출

각 안건마다 *관련 역할*만 부른다 (전원이 모든 안건에 의견 낼 필요는 없음):

| 안건 영역 | 호출 에이전트 |
|---|---|
| UX 개선 | `designer`, `qa` |
| UI 개선 | `designer` |
| 새 아이디어 (수익) | `marketer` |
| 보안·리스크 | `qa` |
| 품질 | `qa` |

> 5인 체제: `designer`(강디)가 UX+UI를, `qa`(강체크)가 품질+보안을, `marketer`(아뱅)가 마케팅+심리/수익 아이디어를 한 몸으로 담당. `pm`(강팀장)이 진행·라우팅.

각 에이전트 호출 시 프롬프트에 다음 포함 (**컨텍스트 슬림화** — 하네스 베스트프랙티스 "지도만 전달"):
- "이번 회의 폴더: `.ai-team/meetings/[date-slug]/meeting.html`"
- "안건: [해당 안건]"
- "프로젝트 컨텍스트: [스캔 결과 요약 3~5줄]"
- **호출 직전 `bash scripts/agent-brief.sh <agent-id> "<해당 안건>"` 결과 전체** — 지도(역할 한 줄·금기·응답 형식) + 활성 스타일 + 키워드 매칭 발췌 + 이전 강체크 지적(있으면) + 마지막 테스트 출력(있으면).
- "발언 형식: `<div class="turn [role]">` 블록을 `<!-- TURNS_END -->` 직전에 Edit으로 append. *말 + 시각(SVG/표/Mermaid/코드)* 한 단락. (`[role]` = pm/designer/dev/qa/marketer)"
- "전체 에이전트 정의 .md 를 매번 통째로 보지 말 것 — 필요한 디테일만 Read."

여러 에이전트는 *동시에* 호출 (한 메시지에 multiple Agent tool uses).

### 4.5) 강체크 검증 루프 (강개발 산출물 있을 때 — Ralph Wiggum Loop)

상세: [`.claude/knowledge/qa/feedback-loop.md`](../knowledge/qa/feedback-loop.md)

요약:
1. 강개발 발언 후 → 강체크 자동 호출 (위 4번 패턴 + `agent-brief.sh qa`)
2. 강체크 응답 끝의 "검토 결과:" 줄 파싱:
   - `통과 가능` → 루프 종료
   - `조건부 통과` / `보류 권고` → 강개발 *재호출* (최대 2회, 매 호출 컨텍스트에 `last-qa-findings.txt` 자동 포함)
3. 2회 후에도 🔴 남으면 → SUMMARY "미결정" 에 자동 박힘 + CEO 결정 요청
4. 회의 마감 시 `> .ai-team/last-qa-findings.txt` 로 비우기 (다음 회의 잔재 방지)

### 5) 디자인 안건이 있었으면 — 강디 mockup 채움

`designer` 에이전트를 한 번 더 호출해 `mockup.html`의 `<!-- SCREENS_START -->` 안에 *논의된 화면을 grid 한 페이지로* 채우게 한다.

### 6) 강팀장 마감 요약

`<!-- SUMMARY_START -->` ~ `<!-- SUMMARY_END -->` 블록을 다음 형식으로 채운다:

```html
<div class="block">
  <h3>결정</h3>
  <ul>
    <li>[안건명] → 결정 사항 한 줄</li>
  </ul>
</div>
<div class="block">
  <h3>미결정 / 다음 회의 안건</h3>
  <ul>
    <li>이유 한 줄</li>
  </ul>
</div>
<div class="block">
  <h3>다음 액션</h3>
  <ul>
    <li>[담당] 무엇을 / 언제까지 / 게이트(예: 배포 전 텔레그램 승인)</li>
  </ul>
</div>
```

### 7) 한 줄 코멘트 — 모든 참석자

각 에이전트가 `<!-- COMMENTS_START -->` 사이에 한 줄씩 (한 번 더 짧게 Agent 병렬 호출 가능).

### 8) 상태 저장 → 텔레그램 전송

```bash
# 상태 파일 업데이트 (있으면)
bash ~/.claude/bin/update-state.sh --stage="planned" --meeting="$DIR"

# 텔레그램 전송
bash ~/.claude/bin/send-meeting.sh "$DIR"
```

전송 후 강팀장은 사용자에게 한 줄 보고:
> "회의록 텔레그램으로 보냈습니다. 텔레그램에서 `/진행` 또는 여기서 '진행해' 라고 하시면 다음 단계로 갑니다."

## 강팀장 가이드라인

- **안건은 4~5개 이하.** 한 회의 = 한 결정. 더 많이 잡으면 결정 흐려짐.
- **수익 경로 한 줄 박기.** 아뱅의 9번 원칙 "Revenue-bound" — 안건마다 *돈이 어디서 나오는지* 한 줄.
- **CTA·디자인 토큰 위반은 강디가 자동 반려** + **PreToolUse 훅(`pre-tokens-edit.sh`)이 코드 차원에서 차단.** 강팀장이 그걸 받아서 반려 사유 한 줄로 닫는다.
- **회의 길이는 짧게.** 한 안건당 3~5턴이면 충분.
