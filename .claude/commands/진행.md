# /진행

CEO가 "진행해" / "다음 단계" / `/진행` 이라고 하면 강팀이 *현재 상태에서 다음 단계*를 자동 실행한다.

`$ARGUMENTS` 있으면 특정 단계 지정 (예: `/진행 배포` → 배포 단계로 점프).

---

## 상태 머신

```
planned → building → testing → fixing → testing → ... → deploy_ready
       ↓                                                       ↓ (게이트: 텔레그램 승인)
   (회의 끝)                                              deploying → promote_ready
                                                                  ↓ (게이트: 텔레그램 승인)
                                                              promoting → done
```

상태는 `.ai-team/state.json` 에 저장:

```json
{
  "stage": "building",
  "meeting": ".ai-team/meetings/2026-05-21-...",
  "decisions": ["..."],
  "actions": [
    {"id": "a1", "owner": "developer", "task": "...", "done": false}
  ],
  "history": [
    {"at": "2026-05-21T10:30:00Z", "from": "planned", "to": "building"}
  ],
  "gates": {
    "deploy": "pending",
    "promote": "pending"
  }
}
```

## 단계별 강팀 행동

### `planned` → `building`
- 강개발이 액션 항목을 코드 변경으로 구현
- 강디가 디자인 시스템 토큰만 사용했는지 자체 검토
- 진행 중 텔레그램으로 진행률 보고 (큰 단계마다)

### `building` → `testing`
- 강체크가 골든 패스 + 엣지 케이스 테스트 (가능하면 자동 테스트, 없으면 수동 시나리오)
- 실패 시 → `fixing` 으로

### `testing` → `fixing` → `testing` (루프, 최대 3회)
- 강개발이 강체크 리포트 보고 수정
- 3회 실패 시 → 텔레그램으로 *사람 개입 요청* 후 정지

### `testing` → `deploy_ready` (✅ 다 통과 시)
- **승인 게이트** — 텔레그램으로 "배포 승인 요청" 메시지 + inline 버튼
- 버튼: `✅ 배포 진행` / `⏸ 보류` / `🔄 한 번 더 테스트`
- 사람 응답 받기 전까지 강팀은 *대기*

### `deploy_ready` → `deploying` → `promote_ready`
- 강개발이 배포 (Vercel/Cloudflare Pages/Netlify 등 — 자격증명은 `~/.claude/team-config/credentials.env`)
- 배포 URL을 텔레그램으로 즉시 보고
- 배포 후 강체크가 라이브 환경 1회 더 검증

### `promote_ready` → `promoting` (✅ 광고 게이트)
- **승인 게이트** — 텔레그램으로 "광고 집행 승인 요청" + 예산·기간 명시
- 버튼: `✅ 광고 집행` / `⏸ 보류` / `💰 예산 조정`
- 승인 후 아뱅가 SecretAD API 호출해서 캠페인 생성

### `promoting` → `done`
- 광고 집행 후 첫 24시간 지표 텔레그램으로 보고
- 강팀장이 회고 시작 (`templates/lesson.md` 복사 후 채움)
- 학습 항목은 inline 버튼으로 강팀 브레인 저장 옵션

## 강팀장 진행 룰

1. **현재 상태 조회**: `cat .ai-team/state.json | jq` 또는 직접 Read
2. **다음 단계 식별** + **승인 게이트 확인**
3. **게이트 미통과면 텔레그램 발송 후 종료** (사람 응답 대기)
4. **게이트 통과면 다음 단계 실행** + 상태 업데이트
5. **단계 끝나면 텔레그램 보고**:
   - 한 줄 요약
   - 현재 상태 머신 위치
   - 다음 액션 / 게이트
6. **3회 실패 시 자동 정지** + 사람 개입 요청
