# /큰지도

CEO 가 "큰지도", "큰지도로 회의", "큰지도 그려줘", "`/큰지도 [주제]`" 라고 하면
일반 `/회의시작` 과 *다른 모드* 의 회의를 연다. **단일 결정이 아니라 전체 그림** 잡기.

`$ARGUMENTS` 가 비어 있으면 강팀장이 *현재 운영 중인 사업 전체*를 큰지도로 그림.
주제가 있으면 그 주제를 *중심에 두고* 큰지도 작성 (예: `/큰지도 콘텐츠 사업`).

---

## 모드 차이

| | 일반 `/회의시작` | `/큰지도` |
|---|---|---|
| 목적 | *한 결정* 내리기 | *전체 그림* 잡고 *공백 영역* 찾기 |
| 안건 수 | 4개 이하 | 영역 단위 (자산·매출·이력·연결선) |
| 산출물 | 결정·미결정·다음 액션 | **자산 맵 + 연결선 + 공백 영역 + 다음 30일 액션** |
| 강팀장 첫 발언 | 안건 ol 리스트 | **자산 매트릭스** (가로 = 사업, 세로 = 자산 종류) |
| 강디 mockup | 화면 와이어 | **자산 흐름 다이어그램** (Mermaid `flowchart`) |
| 텔레그램 보고서 | 결정 박스 | **큰지도 PNG + 공백 영역 우선순위** |

---

## 자동 회의 프로토콜 (강팀장 진행)

### 1) 회의 폴더 생성 — 일반 회의시작과 동일

```bash
bash ~/.claude/bin/start-meeting.sh "큰지도 — $ARGUMENTS"
```

### 3) 멤버 호출 — *순차* (병렬 X), 각자가 *지도에 한 층씩 덧칠*

큰지도는 *덧칠* 이라 *순서가 중요* 함:

1. **강디 호출** — *자산 흐름 시각화* 담당.
   - Mermaid `flowchart LR` 로 사업 간 *화살표* 그리기 (예: "자격증 사이트 합격 사례 → 온라인 강의 플랫폼 마케팅 자료")
   - 화면 mockup X — *플로우차트 1장* 만 핵심
2. **아뱅 호출** — *각 자산의 수익 경로 + 심리 레버* 추가.
   - 각 자산이 *돈으로 어떻게 환원* 되는지 한 줄씩
   - *증거 적층 생태계* 패턴 ([`team-memory/patterns/owned-ecosystem-receipt-stacking.md`](../knowledge/team-memory/patterns/owned-ecosystem-receipt-stacking.md)) 의 3축 적용 점검
   - "이 사업에서 *타인 활동* 이 자산화되고 있나?" 확인
3. **강개발 호출** — *자동화·도구화 가능 영역* 표시.
   - 어떤 자산이 *수동* 으로 누적되고 있나 → *자동화 가능* 한가
   - SEO·SNS 자동 발행·이메일 시퀀스·데이터 export·API 연동
4. **강체크 호출** — *법적·윤리적·보안 리스크* 표시.
   - 타인 자산화 시 *동의·약관* 확인 (개인정보·초상권)
   - 자산 *유출·해킹* 리스크
   - 다크 패턴 가능성

### 4) 강팀장 *덧칠 종합* — 공백 영역 우선순위

모든 멤버 발언 후 강팀장이:

```html
<div class="turn pm">
  <div class="who"><span class="emoji">🎯</span> 강팀장</div>
  <div class="bubble">
    <h4>공백 영역 — 우선순위</h4>
    <ol>
      <li><strong>[영역명]</strong> — 왜 비어 있나 / 채우면 어떤 자산이 생기나 / 채우는 비용</li>
    </ol>

    <h4>사업 간 연결선 — 신규 후보</h4>
    <ul>
      <li><code>자격증 합격생 → 쇼핑몰 스킨 외주</code> — 합격생에게 *본인 사업자 외주 기회* 부여 → 학생도 매출, 본인도 외주 매출, 학생 포폴까지</li>
    </ul>

    <h4>다음 30일 — CEO 결정 필요</h4>
    <ol>
      <li>가장 큰 공백 1개를 *어떤 방식으로* 메울지</li>
      <li>새 연결선 중 *3개월 안에 매출까지 가는* 1개 선택</li>
    </ol>
  </div>
</div>
```

### 6) 전원 한 줄 코멘트 + SUMMARY 채움

일반 회의시작과 동일 절차. 단 SUMMARY 의 **결정** 박스가 *"공백 1개 메우기"* + *"새 연결선 1개"* 두 개로 한정.

### 7) 텔레그램 발송

```bash
bash ~/.claude/bin/send-meeting.sh
```

헤더에 `🗺 큰지도` 표기 (일반 `🎯` 와 구분).

## 강팀장 가이드라인

- **결정은 단 2개** — 일반 회의처럼 한 결정 + 채울 공백 1개 + 새 연결선 1개. 그 이상은 *다음 회의*.
- ***N중 자산* 점검 의무** — 어떤 사업/안건이든 매트릭스의 4칸이 모두 채워질 수 있는지 *자가 점검*.
- **타인 활동 자산화는 *법적 동의 확인* 후만** — 강체크 통과 없이 결정 박스에 박지 않음.
- **시각 우선** — 글만 있는 큰지도는 *큰지도가 아니다*. Mermaid·표·SVG 중 *최소 2개*.
- **단발 회의 안 됨** — 큰지도는 *분기 1회* 이상 갱신. 갱신 시 *이전 큰지도와 비교* 한 줄 (무엇이 채워졌나).
