# /디자인스타일

`$ARGUMENTS` 가 디자인 스타일 번호 (예: `1`, `2`)면 그 스타일을 활성으로 전환한다.
`$ARGUMENTS` 가 비어 있거나 `목록` / `list` 면 현재 활성 + 카탈로그를 보고한다.

## 동작

### 1) 카탈로그 확인 (병렬)

```bash
cat .claude/knowledge/ui-designer/styles/active.txt
ls .claude/knowledge/ui-designer/styles/
cat .claude/knowledge/ui-designer/styles/README.md
```

### 2-A) 번호가 들어왔을 때 — 강디(designer) Agent 호출

`Agent` 툴로 `designer` 서브에이전트 호출 (메인 클로드가 시늉 금지). 프롬프트:

> 활성 디자인 스타일을 #$ARGUMENTS 로 전환해주세요.
>
> 절차 (한 트랜잭션):
> 1. `.claude/knowledge/ui-designer/styles/` 에 `0$ARGUMENTS-*.md` 또는 `$ARGUMENTS-*.md` 파일이 *존재*하는지 확인. 없으면 사용자에게 "스타일 #$ARGUMENTS 가 카탈로그에 없습니다. 등록하시려면 ..." 한 줄 안내 후 종료.
> 2. `.claude/knowledge/ui-designer/styles/active.txt` 를 `$ARGUMENTS` 한 줄로 갱신.
> 3. 해당 스타일 파일을 *처음부터 끝까지* 읽고, 거기 명시된 모든 CSS 토큰을 *현재 레포의* `styles/tokens.css` 에 박는다 (없으면 새로 생성).
> 4. 변경 요약 — 어떤 토큰 키가 추가/변경됐는지 한 줄씩, 보고서는 *실무자 단어 + 풀어쓰기* (CLAUDE.md 보고서 톤 참조).
> 5. "다음 회의·새 화면부터 자동 적용됩니다" 한 줄 보고.

### 2-B) 번호가 없을 때 — 현재 + 카탈로그 보고

다음 형식으로 메인 클로드가 답:

```
현재 활성: 스타일 #N — <이름>  (파일: styles/NN-슬러그.md)

등록된 스타일:
  #1  Crowny Class  — 보라→분홍 그라데이션, Pretendard, 16px 본문
  #N  (다음에 추가될 자리)

전환: /디자인스타일 N  또는  "디자인 스타일 N번 적용해"
새로 등록: styles/NN-슬러그.md 만들고 README 카탈로그에 한 줄 추가
```

## 금기

- **`active.txt` 만 바꾸고 `tokens.css` 안 갱신** 금지. 두 파일은 항상 같은 스타일을 가리켜야 함.
- **존재하지 않는 번호로 전환** 금지. 카탈로그에 없는 번호는 *반려*.
- **메인 클로드가 강디 시늉 금지** — 토큰 작업은 반드시 `designer` Agent 가 한다.
