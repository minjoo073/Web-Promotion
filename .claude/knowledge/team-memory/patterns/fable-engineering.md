# Fable Engineering 작업 방식 — 5가지 규율

> 출처: CEO 가 fable 5 (코딩 에이전트) 와 2개 프로젝트 진행 후 *역분석*해서 추출한 작업 방식 패턴. 강팀 전체가 *코드·결정·배포* 작업 시 의식적으로 적용한다.

> 분류: `patterns/` — 반복 사용 작업 패턴 (도메인 무관, 강개발·강체크·강팀장이 주로 활용)

---

## 1. 불신을 기본값 (계약적 사고)

**무엇**: 외부 입력도, 모델 출력도, *미래의 자기 자신*도 일단 의심한다.

**적용 예**:
- 모델이 준 날짜 → 버리고 *엔진 날짜*로 재조립
- 점수 같은 수치 → 1~5 범위로 `clamp()`
- 경계값 테스트 4종 동시 — `정상 / 매핑 / None / 변조` 입력
- API 응답·DB 조회 결과 → "비어있을 수 있다"·"형식이 바뀔 수 있다" 가정

**강팀 멤버별**:
- **강개발**: try/except 가 아니라 *입력 검증* 을 먼저. 외부 데이터는 *Pydantic·zod 같은 스키마*로 검증 후 통과.
- **강체크**: QA 테스트 케이스에 *항상* 4종 경계 (정상/누락/이상값/공격) 포함.
- **강팀장**: 회의 결정 시 "이 가정이 깨지면?" 질문 1개 의무. CEO 결정 묶음에 포함.

**금기**:
- 모델 출력을 *그대로* 다음 단계로 흘림
- "이건 항상 이 형식이야" 가정으로 검증 생략

---

## 2. 결정을 추적 가능한 객체로 외화 (글로 끄집어내기)

**무엇**: 결정은 *머릿속·채팅* 에 남기지 말고 *글*로. 결정마다 ID (`D1`, `D2`, …, `D7`) 부여, 코드 주석/문서에 **이유 + 비용 + 탈출구** 3개를 같이.

**적용 예시**:
```python
# D3: 결제 API 는 동기 호출 유지
#  이유: 트랜잭션 정합성. 실패 시 사용자에게 즉시 알림 가능.
#  비용: 응답 시간 + 200ms. 동시 요청 100건 한계.
#  탈출구: 트래픽 폭증 시 큐(Redis/SQS)로 전환. D4 참조.
def charge(order):
    ...
```

**강팀 멤버별**:
- **강팀장**: 모든 회의 SUMMARY 의 "결정" 박스에 ID 부여 (`D1 결제 PG 토스`, `D2 무료 사용자 한도 5건`). meeting.html 에 결정 ID 표기.
- **강개발**: 비자명한 분기·구조 선택마다 *코드 주석에 ID + 3종* 필수.
- **강체크**: 리스크 발견 시 결정 ID 인용 — `D3 의 비용 가정이 깨졌습니다 (응답 시간 +600ms)`.

**금기**:
- 결정을 *말로만* (회의·채팅 후 사라짐)
- 결정 이유 없이 코드 변경
- "이유는 알고 있어" 로 생략

---

## 3. 풀기 전에 경계부터 선언 (수렴 스코프)

**무엇**: 작업 *시작 전* 에 **"건드리지 않는 것"** 을 능동적으로 적어 변경 반경을 *먼저 좁힌다*.

스코프 = *발산이 아니라 수렴* 으로 연다.

**적용 예시** — 작업 시작 시 강개발의 첫 응답:
```
[작업: 결제 페이지 카드 컴포넌트 추가]

건드림:
- src/pay/CardForm.tsx (신규)
- src/pay/index.ts (export 추가)
- styles/tokens.css (없으면 생성)

건드리지 않음 (스코프 밖):
- 기존 결제 로직 (src/pay/processor.ts) — D3 결정 유지
- 라우팅 (src/router.ts)
- 결제 PG SDK 버전 — D2 결정 유지
- 다른 페이지의 카드 컴포넌트 — 이번 PR 에서 통합 안 함

영향 반경: 위 3개 파일 + 시각만. 백엔드 API 무관.
```

**강팀 멤버별**:
- **강개발**: 모든 코드 작업 첫 응답에 *건드림/건드리지 않음* 박스 필수.
- **강팀장**: 회의 안건 잡을 때 안건마다 "이번 안건 *밖*의 것" 한 줄.
- **강체크**: 리뷰 시 *경계 선언 안 한* PR 은 반려 (가시성 부족).

**금기**:
- "관련된 거 같이 정리하다 보니..." 식 *발산 PR* (코드 리뷰 비용 폭증)
- 스코프 안 정하고 시작 → 작업 끝에 *우연히 깨진 것들* 발견

---

## 4. 가역성 선호 (얹기 > 갈아엎기)

**무엇**: *갈아엎지 않고* 기존 위에 *얹는다*. 변경 이유는 **별도 커밋** 으로 보존.

가역성 우선순위:
1. 새 파일 추가 (가장 안전)
2. 기존 파일에 새 함수/섹션 추가
3. 기존 함수 *별도 분기* 로 새 케이스 처리
4. 기존 로직 *치환* (가장 위험 — 명시적 결정 필요)

**적용 예시**:
```bash
# ❌ 한 커밋
git commit -m "결제 로직 리팩토링"  # 무엇이 왜 바뀌었는지 모름

# ✅ 분리 커밋
git commit -m "feat(pay): 결제 카드 검증 함수 추가 (D5)"
git commit -m "refactor(pay): processor.charge 가 새 검증 함수 호출 (D5)"
git commit -m "chore(pay): 옛 인라인 검증 코드 제거 (D5 의 옛 케이스, 안전 확인 후)"
```
→ D5 가 잘못된 결정이면 *마지막 커밋만 revert* 하면 옛 상태로.

**강팀 멤버별**:
- **강개발**: 큰 리팩토링은 *3~5개 작은 커밋*으로 분리. 각 커밋이 *독립적으로 revert 가능*.
- **강팀장**: PR 본문에 "rollback plan" 한 줄 필수 — `D5 가 깨지면 commit XXX 1개 revert`.
- **강체크**: revert 가능성 확인 — 한 커밋이 너무 크면 *분리 요청*.

**금기**:
- "이참에 같이 정리" 로 *한 커밋에 5가지* 섞기
- 옛 코드를 *바로 삭제* (먼저 비활성화 + 1주 모니터링 후 삭제)
- *대규모 갈아엎기* (Big Bang Rewrite)

---

## 5. '완료' 의 정의가 넓다 (한 단위 = 동작 + 테스트 + 문서 + 검증)

**무엇**: 코드가 *도는 것* 은 끝이 아니다. **동작 + 테스트 + 문서 현행화 + 검증** 까지가 *한 작업 단위*.

완료 체크리스트:
- [ ] 코드 작동 (수동 클릭 1번)
- [ ] 골든 패스 테스트
- [ ] 엣지 케이스 테스트 2~3개
- [ ] 회귀 체크 (이번 변경으로 깨질 수 있는 기존 기능)
- [ ] 문서 현행화 (README·CLAUDE.md·관련 주석)
- [ ] 결정 ID·이유 기록 (§2 참조)
- [ ] 강체크 검토 통과 (배포 게이트)

**강팀 멤버별**:
- **강개발**: "끝났습니다" 보고 전에 위 체크리스트 7개 모두 ✓. 누락 항목은 *왜 못했는지* 한 줄.
- **강체크**: 위 7개 중 하나라도 빠지면 *배포 보류*. (단 *조건부 통과* 옵션 — CEO 결정 게이트)
- **강팀장**: "다음 액션" 박스에 *완료 정의* 함께 — `결제 카드 컴포넌트 완료 (위 7개 모두 ✓), CEO 배포 승인 대기`.

**금기**:
- "코드는 됐고 테스트는 나중에" (테스트 없으면 *완료 아님*)
- 문서 안 갱신 (다음 사람이 옛 문서 보고 망가짐)
- *조용한 머지* — SUMMARY 에 결정 ID 없음

---

## 메타 패턴 — "외화·경계·가역" 3축

5가지의 공통 정신:

| 축 | 무엇 | 사례 |
|---|---|---|
| **외화** (글로) | 머릿속 → 글 | §2 결정 ID, §5 체크리스트 |
| **경계** (좁히기) | 발산 → 수렴 | §3 스코프 선언, §1 입력 검증 |
| **가역** (안전망) | 한 방 → 단계 | §4 분리 커밋, §1 4종 테스트 |

→ 강팀이 *큰 변경* 을 할 때 이 3축 중 *최소 하나* 가 약하면 사고 확률↑.

---

## 강팀 적용 — 다음 회의·다음 작업부터

1. **강팀장**: 회의 SUMMARY 결정에 ID (`D1`, `D2`…) 부여
2. **강개발**: 작업 첫 응답에 *건드림/건드리지 않음* 박스 + 코드 주석에 `# DN: 이유 / 비용 / 탈출구`
3. **강체크**: PR 리뷰 시 위 7개 체크리스트 + 결정 ID 인용 검토
4. **강디·아뱅**: 디자인·캠페인 결정도 같은 ID 체계 (D5: 메인 컬러 보라 / D6: 첫 채널 인스타)

매 회의 *끝* 에 강팀장이 한 줄 점검:
> "이번 회의 결정에 ID 붙였나? 외화·경계·가역 중 약한 축 없나?"
