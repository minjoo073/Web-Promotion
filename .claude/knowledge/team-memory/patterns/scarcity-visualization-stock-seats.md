## A. 2차 유료 애드온 설계 원리 (앱 전반)

웹네스트 앱은 두 층 구조다(스크립트태그 위젯). **이 사실이 수익·유지보수 설계의 뿌리.**
1. **앱 레이어 = 서버 파일 1벌, 전 몰 자동 반영.** 기능 추가·수정이 스킨 수와 무관 → 팔수록 유지보수 안 늚. 신규 기능은 거의 다 이 레이어.
2. **유료화 = 무료 1차의 "상위판" 사다리.** 무료로 가치를 맛본 사장님이 자연스럽게 업그레이드하게. (매몰비용·자기효능감) 예: 무료 "재고임박 표시(텍스트 배지)" → 유료 "재고 좌석표(움직이는 그리드)".
3. **실측 데이터만 (전자상거래법).** 수요·재고·구매 표시는 전부 실제 데이터에 1:1 연동. 가짜 희소성 금지. → 제약이지만 *진짜라서 더 신뢰*가 되는 무기로 역이용.
4. **발동 조건이 곧 레버.** "전 상품 자동"이 아니라 "사장님이 켠 상품 + 임계값 이하"에서만. 잘못 적용하면 역효과 나는 기능일수록 조건 설계가 핵심.
5. **잠금 UX:** 몰별 settings의 premium 플래그 → 잠긴 기능은 패널에 🔒+안내, 결제 후 해제.

**2차 후보 목록(아뱅 발굴, 실현 검증):** 절약액 미터·최근 본 상품·조회수 증거·웰컴/이탈 팝업·관련상품·1호 구매 레이스·사이즈 쟁탈전·컬러 배틀·녹는 쿠폰·망설임 감지 쿠폰·단골 게이지·가격 기억 배지·드랍 예약·비밀 링크. 묶음: 녹는쿠폰+절약미터+망설임쿠폰="전환 부스터 팩".

## B. 재고 좌석표 (Stock Seats) — 대표 신규 기능

### 한 줄 정의
남은 재고를 **숫자가 아니라 제품 아이콘 그리드**로 보여주고, **실제 팔릴 때마다 한 칸씩 회색으로 사라지게** 해서 구매를 앞당기는 기능.

### 발상의 원천 (CEO)
영화관 좌석 예약에서 찜한 자리가, 펜션 예약에서 찜한 날짜 주변이 **눈앞에서 하나씩 결제돼 없어지는** 걸 보고 급해진 경험. → 그 심리를 상품 상세에 옮김.

### 심리 원리 — 4개 레버
1. **손실회비(loss aversion)** — 회색 칸은 "채워진 것"이 아니라 *비어가는 빈칸*으로 프레이밍해야 한다("내가 머뭇대면 사라진다").
2. **희소성/FOMO** — 남은 칸이 마지막 한 줄일 때 시각적 긴장.
3. **실시간 경쟁** — ★핵심. 영화관 좌석이 무서운 진짜 이유는 "적어서"가 아니라 *"지금 나 말고 누군가도 보고 클릭 중"*. 머무는 동안 눈앞에서 한 칸이 톡 꺼지는 순간을 재현해야 한다.
4. **사회적 증거** — 회색 칸 = "이만큼 남들이 이미 샀다"는 누적 증거.

### ★ 본질 vs 표면 (아뱅 #7, #6)
- 표면: "남은 수를 그림으로." → 정적 회색 그리드. 이건 1차 텍스트 배지와 차별 없음(유료값 약함).
- 본질: **"줄어드는 *움직임*"**. 숫자(추상)를 시각(본능)으로 내리는 것 + *사라지는 동작*. 이 동작이 유료의 값이다.

### 핵심 제약 (안 지키면 역효과/위법)
- **실측 1:1** — 아이콘 수·회색처리 = 실제 카페24 재고/실제 판매. (이미 `stock.php`가 실재고 읽음 → 폴링으로 감지)
- **재고 임계값 이하 + 아이콘 표시상한(약 12~20칸)에서만 노출.** 재고 80개를 아이콘 80개? 화면 깨지고 희소성 0 → 클레임. 고재고에선 자동으로 끄거나 안 띄움.
- 재입고 시 상태 처리(다 회색→다시 컬러 어색함 → "마감/품절" 상태 명시).

### 차별점 = 유료값 (정적 금지)
1. **fade-out**: 머무는 중 stock.php 폴링이 실판매 감지하면 그 칸이 톡 꺼지며 줄어드는 애니메이션(영화관 좌석의 "방금 사라졌다").
2. **마지막 한 줄 펄스**: 남은 컬러 아이콘이 임계(≤3 또는 마지막 줄) 도달 시 *기존 CTA 그라데이션 색 그대로* 맥박. 새 색 안 만듦 → 토큰 유지 + "사야 하는 것" 무의식 연결.
3. **"방금 N개 팔림" 실측 칩**: 폴링 중 감소분만 표시(가짜 금지, 안 줄면 안 띄움). 1차 가짜 구매알림과 달리 100% 실측 → 법적 안전 + 진짜라서 신뢰.

### 효과 매트릭스
| 효과 큼 | 역효과/무의미 |
|---|---|
| 한정판·공구·예약판매(수량/날짜 한정) | 상시 대량재고 생필품 |
| 객단가 높은 망설임 단품 | 옵션 수십 개(어느 옵션 재고?) |
| 재고 1~20개 구간 | 재고 100+ |

### 설명 문구 (설정창 체크박스)
- 짧게: **"남은 재고를 아이콘으로 시각화 — 팔리면 한 칸씩 사라져 구매를 재촉합니다 (실재고 연동)."**
- 길게: "남은 재고를 숫자 대신 제품 아이콘으로 보여주고, 팔릴 때마다 한 칸씩 사라집니다. 영화관 좌석처럼 '지금 안 사면 없어진다'는 느낌으로 구매를 앞당깁니다. 실제 재고에 연동되어 거짓표시가 없습니다."

### 기능명 후보
**재고 좌석표 (Stock Seats)** ←추천(발상 직결, 고객도 '좌석=줄어든다' 학습됨) · 사라지는 재고(Melting Stock) · 마지막 자리(Last Spots).

## C. 맞춤 아이콘 유료 요청 (Custom Stock Icon) — 부가 유료 옵션

### 정의
재고 좌석표의 칸 아이콘을 기본 제공 아이콘 대신 **그 쇼핑몰·상품에 꼭 맞는 전용 디자인**으로 제작해주는 **별도 결제 요청** 기능.

### 설명 문구 (설정창/안내)
> 재고 좌석표 아이콘을 우리 브랜드·상품에 맞춘 **전용 디자인**으로 만들고 싶다면, 버튼 한 번으로 맞춤 제작을 요청하세요(유료). 기본 아이콘으로도 작동하지만, 브랜드 톤에 맞춘 전용 아이콘(예: 화장품=립스틱, 의류=옷 실루엣)은 더 눈에 들어오고 고급스러워 구매 신뢰·전환에 도움이 됩니다.

### 원리
- **자기맞춤화 + 브랜드 일관성 → 체감가치↑.** 범용 아이콘은 "기능", 전용 아이콘은 "우리 것" → 사장님 지불의향↑.
- **추가 수익원(서비스 매출).** 기능 사용료와 별개의 디자인 제작 단가. (강디1/강디2 제작 → 적용)
- **사다리 한 칸 더.** 무료 배지 → 유료 좌석표 → 유료 맞춤 아이콘. 단계마다 더 깊은 결제.

## D. 일반화 — 다른 기능 설계에도 쓰는 원리
1. **추상을 본능으로 내려라**(숫자→시각, 텍스트→움직임).
2. **"상태"보다 "변화의 순간"을 보여줘라**(머무는 중 눈앞에서 바뀌는 1초가 전환을 만든다).
3. **실측 제약을 신뢰 무기로 역이용**(가짜는 위법이자 약함, 진짜는 강함).
4. **무료→유료→맞춤 사다리**로 결제 깊이를 단계화.
5. **역효과 나는 기능일수록 "발동 조건"이 본체**(임계값·상한·on-off).
