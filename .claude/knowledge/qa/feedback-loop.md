# 강체크 검증 루프 (Ralph Wiggum Loop)

> 하네스 엔지니어링 베스트프랙티스의 핵심 패턴 — *검증 결과를 다음 호출 컨텍스트에 자동 주입*해 에이전트가 스스로 고치게 한다. OpenAI Codex 가 "린터 에러를 에이전트 컨텍스트에 직접 주입" 으로 효과를 본 것이 출처. 강팀에선 강체크가 그 역할을 한다.

---

## 루프 설계

```
강개발 산출물
    │
    ▼
강체크 검증 ──── 🟢 통과 ──── ▶ 결정 박스
    │
    🔴 / 🟡 발견
    │
    ▼
.ai-team/last-qa-findings.txt 에 적재
    │
    ▼
강개발 재호출 (자동, 최대 2회)
  ├─ 첫 줄에 "[이전 강체크 지적]" 박힘 (agent-brief.sh 가 자동 포함)
  └─ 강개발은 그 지적부터 해결
    │
    ▼
강체크 재검증
    │
    ▼ (2회 후에도 🔴 남음)
회의 마감 "미결정" 섹션에 자동 기재 → CEO 결정 요청
```

---

## 회의시작 프로토콜 §4.5 — 검증 루프

강개발이 발언을 마치면 자동으로:

### 1) 강체크 호출 (Agent 툴, qa)
프롬프트에 포함:
- "이번 회의 폴더 + 안건"
- `bash scripts/agent-brief.sh qa "<안건>"` 결과 — 지도·발췌·이전 지적·마지막 테스트 출력
- "강개발의 직전 발언 발췌 (turn dev 마지막 1개)"
- "체크리스트: 골든패스·엣지·회귀·디자인 토큰 + 보안·프라이버시·법규·다크패턴"
- "발견을 `meeting.html` 자기 턴에 박고, 🔴/🟡 가 *하나라도* 있으면 `.ai-team/last-qa-findings.txt` 에 *덮어쓰기*"

### 2) 자동 재호출 트리거
강체크 응답 끝의 "**검토 결과: ..**" 줄을 파싱:
- `통과 가능` → 루프 종료, 결정 박스로
- `조건부 통과` 또는 `보류 권고` → 강개발 재호출 (아래 3 단계)

### 3) 강개발 재호출 (최대 2회)
프롬프트에 포함:
- "이전 강체크 지적: `.ai-team/last-qa-findings.txt`" (전체 인용)
- "위 지적 *순서대로* 한 줄씩 해결안 박고 코드 수정. 안 되는 지적이 있으면 *왜 못 하는지* 한 줄로 명시 (그게 다음 미결정 후보)"
- "수정 후 강체크 재호출됨 — 같은 지적 다시 나오면 시스템 실패"

### 4) 강체크 재검증 → 위 1) 반복 (카운터 +1)

### 5) 카운터 ≥ 2 + 여전히 🔴
- `.ai-team/last-qa-findings.txt` 의 🔴 항목을 회의 `<!-- SUMMARY_START -->` 의 "미결정 / 다음 회의 안건" 섹션에 자동 박음
- 강팀장이 마감 요약에서 CEO 에게 *결정 요청* (예: "이 리스크는 비즈니스 가치 때문에 *수용* 할지 *완전 차단* 할지")

## `.ai-team/` 휘발성 파일 규약

| 파일 | 누가 쓰나 | 누가 읽나 | 수명 |
|---|---|---|---|
| `last-qa-findings.txt` | 강체크 | 다음 강개발 호출 (`agent-brief.sh` 가 자동 주입) | 회의 마감 시 비움 |
| `last-test-output.txt` | PostToolUse 훅 (npm test/pytest 종료 후) | 다음 강체크·강개발 호출 | 다음 테스트 실행 시 덮어씀 |

→ `.gitignore` 에 `.ai-team/last-*.txt` 추가 (회의록 폴더 `meetings/` 는 커밋, 휘발성 파일은 X).

## 안티패턴 — 절대 하지 말 것

- **메인 클로드가 강체크 응답을 *대신 해석*** — "강체크가 OK 한 거 같으니 넘어가자"는 시늉 금지. 응답 끝의 "검토 결과:" 줄을 *문자 그대로* 파싱.
- **강체크가 통과 줬는데도 강개발 재호출** — 검증을 통과한 것을 *예방적으로 다시 손보면* 회의가 무한 루프.
- **재호출 카운터 무한 증가** — 같은 지적이 3회째 나오면 시스템이 무한 루프에 빠진 것. 강제 종료 + 미결정으로.
- **`last-qa-findings.txt` 를 사람이 직접 편집** — 다음 회의 컨텍스트에 *유령 지적* 이 섞임. 강체크만 쓴다.

## 회의 마감 시 정리

강팀장이 회의 마감 직전:
```bash
# 휘발성 파일 비우기 (다음 회의에 잔재 남기지 않기)
> .ai-team/last-qa-findings.txt
```
단, *해결 안 된 🔴* 가 있으면 그 항목을 `meeting.html` 의 SUMMARY 미결정에 *이미 박은 뒤* 비우는 것.
