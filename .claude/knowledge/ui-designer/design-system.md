# 강팀 디자인 시스템 — 진입점

> **활성 스타일 포인터.** 실제 토큰·컴포넌트·규칙은 `styles/` 디렉토리의 *번호 매겨진 스타일 파일* 한 곳에 있다.

## 현재 활성

`styles/active.txt` 에 적힌 한 줄(예: `1`) 이 현재 활성 스타일 번호이며,
그에 대응하는 파일이 *그 프로젝트의 디자인 시스템 진본*이다.

기본값: **스타일 #1 — Crowny Class** ([`styles/01-crowny-class.md`](./styles/01-crowny-class.md))

## 스타일 카탈로그·전환 방법

전체 카탈로그와 전환 방법은 [`styles/README.md`](./styles/README.md) 참고.
요약:

- "디자인 스타일 N번 적용" 자연어 → 강디가 `active.txt` 갱신 + `styles/tokens.css` 재작성
- `/디자인스타일 N` 슬래시 커맨드 — 같은 결과
- 새 스타일 추가는 `styles/NN-슬러그.md` 파일 만들고 카탈로그 표에 한 줄 등록

## 강디가 매 회의 첫 턴에 *반드시* 한 줄 박을 것

```
현재 활성 디자인 스타일: #N — <이름>  (출처: styles/NN-슬러그.md)
```

이 줄이 빠지면 강팀장이 회의를 *반려*한다. 디자인 작업이 *어떤 스타일 위에서* 굴러가는지 모르고 시작하는 회의는 무효.
