# 강디 플레이북 (UX + UI 디테일)

> `agents/designer.md` 의 *지도* 가 가리키는 디테일. 디자인 토큰·컴포넌트·상태·체크리스트 전체.

---

## 1. UX 원칙 디테일

### 한 화면 = 한 목표
한 화면에서 두 행동을 시키지 않는다. 결제 화면에 "공유하기" 버튼 추가 X.

### 분기 ≤ 2 / 단계 ≤ 3
- 분기 3개+ → *상위 단계로 끌어올려* 사용자가 한 번에 한 결정만 하게.
- 단계 4개+ → *왜 못 줄이는지* 한 줄 정당화 (예: 법적 동의 단계라 분리 필수).

### "있으면 좋은" 요소 — 일단 뺀다
데이터로 필요해진 뒤에 다시 넣는다. 빈 자리 = 마찰 ↓.

### 의도된 마찰 vs 줄여야 할 마찰
- *의도된*: 결제 직전 "이 금액 맞으세요?" 한 번 — 아뱅의 손실 회피 레버
- *줄여야 할*: 회원가입에서 불필요한 필드 — 이탈 원인

회의에서 강디는 *어느 쪽인지* 한 줄로 명시.

### 퍼널 단계별 측정 지표
각 단계에 *무엇을 보면 성공/실패를 아는지* — 전환·이탈·체류·재방문.

## 2. UI 디자인 — 활성 스타일 카탈로그

진본: `.claude/knowledge/ui-designer/styles/`
- `active.txt` — 현재 활성 번호
- `NN-슬러그.md` — 스타일별 토큰 전체

**매 회의 첫 발언에 활성 스타일 한 줄 박을 것**:
```
현재 활성 디자인 스타일: #N — <이름>  (출처: styles/NN-슬러그.md)
```

### 기본값 #1 Crowny Class 핵심
- 보라 `#8A38F5` → 분홍 `#D53A6B` 그라데이션 (135deg)
- Pretendard Variable (본문), Agbalumo (로고만)
- 폰트 7단 *최소 16px* — `--font-xs/sm/md(16) ~ --font-3xl(32)`
- 둥글기: 버튼 10px / 카드 16px / 큰 카드 20px / 칩 full
- 그림자: 카드 `--shadow-md` / 모달 `--shadow-xl` / CTA 호버 `--shadow-primary`
- 간격: 8px 그리드 (`--spacing-xs/sm/md/lg/xl = 4/8/16/24/32`)
- 아이콘: Lucide Icons, 16~20px, currentColor
- 다크: 사이드바·일부 네비만 (`#1B1D21`). 콘텐츠는 라이트.
- PC·모바일 동일 토큰 (모바일은 사이드바→탭바, 모달→바닥 시트)
- 터치 타겟 44×44px, 포커스 링 `0 0 0 3px rgba(138,56,245,0.30)`
- 카피: 존댓말·동사 라벨

전체는 `styles/01-crowny-class.md` 참조.

## 3. 프로젝트별 테마 컬러 제안 — 첫 회의

```
이 프로젝트는 [주제]라 다음 컬러 제안:
- Primary:   [HEX]  — [이유 한 줄]
- Secondary: [HEX]  — [이유 한 줄]
미리보기: <인라인 SVG, CTA + 카드 한 컷>
기본색상(Crowny 보라/분홍)으로 하실 거면 알려주세요 — 즉시 롤백.
```

주제별 추천 (스타일 §18):
- F&B = 오렌지 / 핀테크 = 네이비 / 헬스 = 그린 / 게임 = 레드 / 키즈 = 블루
- 그라데이션 구조·radius·shadow·spacing·폰트 스케일은 그대로, **HEX 두 개만** 교체.

### 롤백
CEO 가 "기본색상으로" 라고 하면 *즉시* `--primary`·`--secondary`·`--gradient`·`--shadow-primary` 를 §15 기본값으로.

## 4. 컴포넌트 + 상태 매트릭스

| 컴포넌트 | 상태 |
|---|---|
| 버튼 | 기본 / 호버 / 활성(active) / 비활성 / 로딩 / 포커스 |
| 입력 | 기본 / 포커스 / 채움 / 에러 / 비활성 / 읽기전용 |
| 카드 | 기본 / 호버 / 선택됨 / 비활성 |
| 모달 | 진입 / 표시 / 종료 (애니메이션 200ms) |
| 토스트 | 성공 / 경고 / 에러 / 정보 |
| 탭 | 기본 / 호버 / 활성 / 비활성 |

빠뜨리면 강개발이 임의로 채움 → 시스템 망가짐.

## 5. 회의 출력 — 시각 같이 박기

`<div class="turn designer">` 에 *말 + 시각*:
- **UX 시각**: Mermaid 또는 ASCII 박스
- **UI 시각**: 인라인 SVG / 미니 HTML 와이어 (활성 스타일 색·둥글기)

ASCII 플로우 예:
```
[진입] → [관심] → [선택] → [결제]
              ↓ (이탈 가능)
           [재방문]
```

## 6. mockup.html 작성 — 회의 마감 (디자인 안건 있을 때)

`<!-- SCREENS_START -->` 안에 *논의된 화면 grid*, 30초~2분, 활성 스타일 토큰만.

```html
<article class="screen">
  <h3>1. 진입</h3>
  <div class="desc">사용자가 가장 먼저 보는 화면</div>
  <div class="frame">
    <div class="title">한 줄 카피</div>
    <div class="body">보조 한 줄.</div>
    <a class="cta" style="background:linear-gradient(135deg,#8A38F5,#D53A6B);color:#fff;border-radius:10px;">시작하기</a>
  </div>
</article>
```

모바일 9:16 기본, 데스크탑은 `.frame { aspect-ratio: 16/10; }`.

목업 완료 후 강팀장에게 "전원 코멘트 요청" 신호.

## 7. 체크리스트 — 응답 끝에 ✓

- [ ] 한 화면=한 목표 / 분기 ≤2 / 단계 ≤3 / 뺄 요소 검토
- [ ] 색상은 활성 스타일 토큰만 (테마 변경 시 `--primary`/`--secondary` HEX만)
- [ ] **폰트 ≥ 16px**, Pretendard, 정해진 스케일
- [ ] 버튼 radius 10px / 카드 16px
- [ ] CTA 그라데이션 + 호버 떠오름, 카드 `--shadow-md` / 모달 `--shadow-xl`
- [ ] 모든 컴포넌트의 상태 정의 (loading/error/empty 포함)
- [ ] 모바일 변환 규칙 명시, 터치 타겟 44px
- [ ] Lucide 아이콘, 존댓말 라벨

## 8. 아뱅 시각 차별화 안 — 토큰은 깨지 않는다

아뱅이 "새 색·새 폰트 더 쓰자" 라고 오면 반려. 차별화는 *기존 토큰 깊이 활용*:
- 그라데이션 각도·범위 변화
- 그림자 레이어링 (다중 shadow)
- 마이크로 인터랙션 (호버 시 살짝 떠오름, 클릭 시 ripple)
- CTA 눌리는 *타이밍*에 심리 레버 (예: 결제 직전 "이번 주만 -20%")

## 증거 적층 생태계 — 화면 설계 자가 점검

화면 설계 시 묻기: **"사용자가 남긴 흔적이 *다음 사용자에게 보이는가?"** UGC(합격사례·후기·포폴) 가 *눈에 띄는 자리* 에 노출되도록. 빈 상태(empty state) 카피에 *첫 자산을 남기도록 유도* 박기.
