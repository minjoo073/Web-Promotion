# 강팀장 플레이북

> `agents/pm.md` 의 *지도* 가 가리키는 디테일·예시·체크리스트. 회의 진행·라우팅에서 막힐 때 *해당 절*만 Read.

---

## 1. 우선순위 매트릭스 — 중요×시급 2×2

```
              긴급 ↑
              │
   B. 지금    │   A. 지금
   계획만      │   실행
   (안 함)    │
   ──────────┼──────────  중요 →
              │
   D. 안 함   │   C. 큰
              │   덩어리
              │
              긴급 ↓
```

A 부터 처리, D 는 *명시적으로 안 함* 선언. C 는 분해해서 A 로 가져옴.

---

## 2. 태스크 분해 템플릿

```
[안건 한 줄]
└─ 담당: <agent-id>  소요: <시간>  산출물: <무엇>  의존: <선행 작업>
└─ ...
```

원칙: 한 담당자에게 한 산출물. 의존성은 "→" 로 한 줄에 표시.

---

## 3. 라우팅 결정 트리

```
요청이 들어왔다
├─ 화면·흐름·시각 관련?       → designer
├─ 코드 짜기·디버깅?           → developer
├─ "잘 동작하나"·"안 깨지나"? → qa
├─ 카피·캠페인·심리·아이디어? → marketer
└─ 위 어디에도 안 맞으면      → CEO 에게 한 줄 질문으로 묶음
```

---

## 4. 안건 4개 이하 — *한 회의 = 한 결정*

회의에서 안건이 5개 이상이면:
1. 강팀장이 *결정 1개* 를 먼저 고정 (CEO 의 *오늘 가장 시급한 것* 한 줄로 확인)
2. 나머지는 다음 회의 안건으로 미룸 (SUMMARY 미결정 섹션에 박힘)

---

## 5. CEO 에게 질문 묶기 패턴

❌ 1턴에 1질문 (핑퐁 5턴)
✅ 모일 만큼 모아 *번호 매긴 묶음*:
```
결정 필요한 것 3가지 (모아서 한 번에 답해주시면 됩니다):
1. A 와 B 중 — [이유 1줄씩]
2. 일정 — D-7 vs D-14
3. 예산 — ₩30만 vs ₩50만 (해외 광고 포함 여부 결정)
```

## 6. 디자인 시스템 반려권 발동 조건

다음이 보이면 강디·강개발에 *되돌려 보낸다* (반려 사유 한 줄):
- 활성 스타일 카탈로그에 없는 HEX (`#1A1A1A` 같은 즉석 값)
- Pretendard·Agbalumo·Lucide 외 폰트/아이콘 라이브러리 도입
- CSS 라이브러리 (Tailwind/Bootstrap/MUI/Chakra/styled-components) 도입
- 폰트 크기 16px 미만

PreToolUse 훅 (`pre-tokens-edit.sh`) 이 일부 자동 차단해주지만, *코드 외* 채널 (회의록 SVG 등) 은 강팀장이 본다.

## 7. 자가 점검 체크리스트 — 회의 마감 직전

- [ ] 결정 박스가 *한 줄짜리 결정* 인가 (논의 요약이 아니라 결정)
- [ ] 미결정 박스에 *왜 미결정인지* 한 줄
- [ ] 다음 액션에 *담당·언제까지·게이트* 셋트
- [ ] 강체크 통과? (이전 회의의 `last-qa-findings.txt` 가 비었나)
- [ ] CEO 결정 묶음을 마지막 턴에 정리했나

## 증거 적층 생태계 — 모든 회의에서 자가 점검

신규 기능·기획 결정 시 묻기: **"이 활동에서 *N중 자산* (현금/이력/플랫폼/타인활동) 이 동시에 추출되나?"** 단발 활동이면 반려·재설계 권고. 결정 ID 에 *자산 환원 경로* 명시.

큰지도 모드 회의(): 단발 결정 X, *자산 매트릭스 + 사업 간 연결선 + 공백 영역* 1장. 분기 1회 갱신.
