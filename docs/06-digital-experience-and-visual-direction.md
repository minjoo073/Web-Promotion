## 0. ★ rev.2 변경 핵심 — 하이브리드 발굴 era (CEO 확정)

> **핵심 모티프 = 한국 후기웹(2005~2012, 싸이월드 미니홈피) / 보조 질감 = 글로벌 후기웹(Tumblr·캠코더).**

- **왜 한국 후기웹이 핵심인가:** KiiiKiii 1차 팬의 *진짜 정서 기억*은 Tumblr가 아니라 **싸이월드 미니홈피**다. "복원"이 한국 정서에선 *미니홈피 복원*("내 사진첩 복구 중" — 실제 2022 싸이월드 부활 때 헤드라인)으로 직역된다. [Source: 한국일보 "내 사진첩은 복구 중"; 나무위키 싸이월드]
- **싸이월드 실제 UI(리서치 확정):** 미니홈피 중앙 = 최근 게시물·**미니미**(아바타·기분표시)·**미니룸**·일촌평 / 우측 메뉴 = 프로필·게시판·사진첩·다이어리·방명록 / **파도타기**(일촌 미니홈피로 점프 이동) / **도토리**(acorn = 스킨·BGM·미니룸 인테리어·폰트 구매 가상화폐) / 좌상단 **today·total 방문자 카운터** / **BGM 플레이어**(감성곡, 한국 웹문화에 각인). [Source: 나무위키 싸이월드/서비스; Cyworld — Wikipedia; 파이낸셜뉴스 2021 UI]
- **보조 질감(레이어로만):** Tumblr 무한스크롤·리블로그 화살표·캠코더 인터레이스·JPEG 번짐은 *질감/동작* 레이어로 얹기. 골격은 미니홈피.
- **가드레일 동일:** indie sleaze 퇴폐·음주 코드 시각 배제(막내 안전선). §3-6 "촬영 질감 OK / 퇴폐 맥락 차단" 표 그대로 유지.

## 1. UX / IA — 발굴형 흐름 3단계 (한국 후기웹 메타포로 재서술)

흐름 한 줄: **죽은 미니홈피·끊긴 BGM에 걸려든다 → 도토리·파도타기로 복원에 기여한다 → 내 미니룸이 공식에 편입되고 멤버가 일촌평으로 받아친다.**
(감상형이면 "본다→좋아요→나간다". 발굴형은 "걸린다→판다→다시 끌어온다". 이탈마다 *행동* 요구. ≤3 분기.)

```
[유입: TikTok 챌린지 / 흘러나온 죽은 미니홈피 링크]
            │
            ▼
 ① 발굴 (DISCOVER)  ──이탈①──▶ "그냥 닫힌 옛날 사이트인 줄"
   = 죽은 미니홈피·정지된 BGM 발견·접속
            │
            ▼
 ② 복원 (RESTORE)   ──이탈②──▶ "혼자 차오르는 게 더디다"
   = 도토리/파도타기로 복원 기여·영역 언락
            │
            ▼
 ③ 공동운영 (CO-RUN)──이탈③──▶ "내가 꾸민 게 안 보인다"
   = 내 미니룸 공식 편입 + 멤버 일촌평("RE:") 응답
            │
            ▼
 [재유입 루프: 멤버 일촌평 → 파도타기로 다시 발굴]
```

### 1-1. ① 발굴 (DISCOVER) — "이 미니홈피, 살아있는데?"
- **목표 행동:** "삭제됨/접속 불가"로 보이는 *죽은 미니홈피*(끊긴 BGM·깨진 사진첩 썸네일·먹통 방명록)에 접속해 *복구를 시도*(클릭·BGM 재생 버튼 누르기)하게 만든다. 첫 발굴 = 첫 클릭.
- **흐름:** 깨진 미니홈피 진입 → "복구 시도"(BGM 재생/사진첩 열기) 단일 CTA → dashboard 언락. **분기 1개**. 가입·튜토리얼 마찰 0.
- **이탈 원인①:** 그냥 닫힌 옛날 사이트로 오인. → *대응:* "깨짐"과 "고장"의 시각 구분 — today·total **방문자 카운터가 0에서 똑딱 올라가고**, 미니미가 커서에 반응해 꿈틀, 정지된 BGM 막대가 *재생 직전*으로 떨림. "버려진 게 아니라 잠든 미니홈피".
- **측정 지표:** 죽은 미니홈피 진입 → "복구 시도" 클릭 **전환율(CTR)**. 첫 클릭까지 시간(time-to-first-dig).

### 1-2. ② 복원 (RESTORE) — "도토리 넣으니까 BGM이 돌아온다"
- **목표 행동:** *복원 기여 행동*(리블로그/공유/태그 = 이번 era의 "도토리 넣기")으로 복원 진행률을 올린다. 깨진 영역(끊긴 BGM·미공개 캠코더·번진 셀카·다이어리 한 줄)이 한 칸씩 언락되는 즉각 보상.
- **흐름:** 깨진 사진첩/BGM hover·탭 → "이거 복원하기"(도토리 모티프) → 진행률 +x% → 인접 영역 1칸 복원. **분기 2개**(개인 즉시복원 / 집단 임계치 대기). **파도타기**로 다른 팬의 복원 중 미니룸으로 점프해 추가 기여 가능.
- **이탈 원인②:** 혼자선 더뎌 "내 도토리 하나는 의미 없다" 무력감. → *대응:* 개인 행동에 *즉시* 미세 보상(내가 만진 1칸 즉시 복원) + 집단 진행률은 *별도 바*. "내 기여(즉시)" + "전체 복원(누적)" 이중 피드백.
- **측정 지표:** 1인당 평균 복원 액션 수, 임계치별 언락 도달률, 파도타기 점프 후 추가기여율.

### 1-3. ③ 공동운영 (CO-RUN) — "내 미니룸이 공식에 걸리고, 멤버가 일촌평을 남겼다"
- **목표 행동:** 팬이 *꾸민/복원한 미니룸·폴더*가 공식 dashboard에 편입되고, **멤버가 "일촌평(=RE:)"으로 응답** → 일방숭배 아닌 왕복(티키타카=파도타기의 왕복성). 팬=공동운영자.
- **흐름:** 내 복원/미니룸 게시 → 공식 편입 큐 → 멤버 일촌평 알림 → 파도타기로 재발굴. **분기 1개**(편입됨/대기).
- **이탈 원인③:** "내가 꾸민 게 어디 갔는지 안 보임"(기여 비가시). → *대응:* 모든 복원 1칸에 *복원자 핸들* 워터마크("복원 by @___" = 싸이 "일촌평 남긴 사람"의 구조). 익명 군중이 아니라 *이름 있는 일촌*. 멤버 일촌평은 별도 색·모션.
- **측정 지표:** 팬 미니룸 공식 편입 수, 멤버 일촌평 후 24h 재유입, 복원 100% 공동달성률.

> **왜 3단계인가:** docs/05의 6단계 캠페인 타임라인을 *팬 한 명의 체감 동선*으로 압축. 캠페인은 시간순, UX는 행동순. 매핑 — 발굴=0·1단계, 복원=2·3단계, 공동운영=3·5단계.

## 2. 핵심 인터랙션 3개 — 한국 후기웹 매핑 + 상태별 동작(loading·error·empty·disabled·focus)

> 핸드오프는 픽셀이 아니라 *동작 + 상태*. 상태 누락 금지. 본문 폰트 ≥16px(질감은 장식 레이어, 가독성 별개).

### 2-1. 인터랙션 A — 복원 진행률 바 (RESTORE METER) ★핵심 UI
캠페인의 심장. **싸이 today·total 방문자 카운터 + BGM 로딩 막대**를 복원 게이지로 재해석. 일반 progress가 "기다림"이라면 이건 "참여의 누적".

- **기본 동작:** 두 겹 — (a) *내 기여*(즉시, 미세) + (b) *전체 복원*(팬덤 누적, 큰 게이지). 임계치(25/50/75/100%)마다 영역 언락. 카운터가 today/total처럼 *똑딱똑딱* 오르는 한국 후기웹 모티프.
- **시각 방향:** 매끈한 채움 막대 ✕ → 끊긴 BGM 막대가 *한 칸씩 차며 재생 준비*되는 느낌 + (보조) 캠코더 디인터레이스로 선명해짐.
- **상태:**
  - `loading` — 집계 중: 카운터가 *지직거리며 값 미확정*. 스켈레톤 = "아직 복원 안 된" 회색 스캔라인(스켈레톤=복원 전 상태와 시각 통일). `aria-busy="true"` + 시각가림 텍스트 "복원률 불러오는 중". [Source: NN/g Skeleton Screens; Adrian Roselli]
  - `error` — 집계 실패: "복원 신호 끊김 / 다시 시도" — *에러도 세계관 언어*("접속이 또 끊겼다"). 마지막 성공값 유지(0 리셋 금지 = 무력감 차단).
  - `empty` — 0%(시작 직후): 완전히 깨진 미니홈피 + "아직 아무도 복원하지 않음 — 첫 방문자가 되라"(today 카운터 0). 빈 상태가 *초대장*.
  - `disabled` — 임계치 미달 잠금 영역: 자물쇠 아닌 *"아직 깨진 채"*(미복원 사진첩 썸네일 깨짐) + "x% 더 필요".
  - `focus` — 키보드 포커스 시 게이지/언락 영역 또렷한 포커스 링(3:1 대비, 노이즈가 포커스 가리지 않게). [Source: WCAG 비텍스트 3:1]

### 2-2. 인터랙션 B — "깨진 → 복원" 상태 전환 (DECAY → RESTORE)
*화면의 기본 상태가 깨짐*이라는 제1제약의 실행. **끊긴 BGM 재생 복원 · 깨진 사진첩 썸네일 · 빈 미니룸**이 행동으로 살아남.

- **기본 동작:** 깨진 요소(정지된 BGM 막대·이미지 깨짐·텅 빈 미니룸·잘린 다이어리) → 팬 행동(도토리/리블로그) → *복원*: BGM이 다시 재생되고, 사진첩 썸네일이 프로그레시브 JPEG처럼(흐릿→선명, 위→아래) 풀리고, 미니룸에 미니미·가구가 채워짐.
- **시각 방향:** "복구 중" = 2000년대 *프로그레시브 JPEG 로딩* + *BGM 버퍼링 막대*. 그 시대의 실제 로딩 경험이라 노스탤지어와 기능이 일치. (보조: GIF 재시작·캠코더 디코딩)
- **상태:**
  - `loading` — 복원 처리 중: 프로그레시브 디코딩 + BGM 버퍼링 애니메이션. 낙관적 UI(일단 복원된 듯 보이고 실패 시 롤백).
  - `error` — 복원 실패: 요소가 *다시 깨짐*으로 롤백 + "복원 실패, 다시"(세계관상 "또 손상됐다"). 데이터 손실 없이 재시도.
  - `empty` — 복원할 게 없는 폴더/미니룸: "이 미니룸은 비었다 / 다른 죽은 미니홈피를 찾아라"(404 DNA 계승, 막다른 길→다음 발굴).
  - `disabled` — 이미 100% 복원: 복원 버튼 사라지고 "복원 완료 ✓ by @___". 더 누를 것 없음 명확.
  - `focus` — 깨진 요소 포커스 시 "이걸 복원" 노출 + alt/aria로 "손상된 항목, 복원 가능" 고지.

### 2-3. 인터랙션 C — dashboard = 미니홈피 (메인 + 다이어리/방명록 + 파도타기)
**골격 = 싸이 미니홈피**(중앙 최근글·미니미·미니룸 / 우측 메뉴 프로필·사진첩·다이어리·방명록 / 좌상단 today·total). **보조 = Tumblr 무한스크롤·리블로그 화살표**(타임라인 동작 레이어). [Source: 나무위키 싸이월드/서비스; Tumblr social history]

- **기본 동작:** 미니홈피 메인에 다이어리·방명록·사진첩 진입. 방명록/리블로그가 곧 복원 연료(→A로 연결). **파도타기 = 다른 팬의 복원 중 미니룸으로 왕복 점프**(TiiiKiii 왕복=티키타카의 시각화, Tumblr "reblog 화살표↻"가 이 점프의 보조 모티프).
- **시각 방향:** 싸이의 *단정한 미니홈피 프레임*(좌측 미니룸/우측 메뉴 골격) + 캠코더/플래시 질감 콘텐츠. "단정한 골격 + 날것 콘텐츠"의 의도적 충돌이 grime을 만든다.
- **상태:**
  - `loading` — 다음 묶음/방명록 로딩: "load more"가 *2000년대식 점 깜빡임* 스피너 + 스켈레톤(스캔라인 회색). 스크롤·파도타기 위치 보존.
  - `error` — 피드/방명록 로딩 실패: "미니홈피 연결 끊김 / 새로고침" — 이미 로드된 항목 유지(전체 날림 금지).
  - `empty` — 태그/방명록 결과 0: "이 태그로 복원된 미니룸이 아직 없음 — 네가 첫 방명록을 남겨라"(빈 검색→생성 유도).
  - `disabled` — 비로그인 리블로그/방명록 시도: 버튼 비활성 + "일촌(발굴자)으로 합류해야 복원 가능"(가입을 *일촌 맺기*로 리프레임). 회색 + not-allowed.
  - `focus` — 키보드 타일/메뉴 이동: 항목 단위 포커스(reblog·방명록·tag 탭 순서), 무한스크롤/파도타기가 키보드 사용자를 가두지 않게 스킵 링크.

## 3. 비주얼 시스템 방향성 (CEO 디자인용 브리프)

> 임의 HEX·폰트 지정 ✕ → *방향*만. CEO가 이 방향 안에서 픽셀 확정.

### 3-1. 무드 / 질감 / 모티프
- **무드:** "삭제 직전 누군가의 *미니홈피*를, 복구 프로그램이 한 줄씩(끊긴 BGM·깨진 사진첩) 되살리는 중." 향수 + 복구의 긴장. 매끈한 완성이 아니라 *복구 진행 중의 불완전*이 정서 본체.
- **질감(핵심=한국 후기웹):** 미니홈피 스킨·미니룸·미니미·도토리·today/total 카운터·BGM 플레이어·방명록 위젯·초기 블로그 스킨/카운터. **(보조=글로벌):** 캠코더 인터레이스/타임스탬프, 플래시 적목·핫스팟, JPEG 번짐, 무한스크롤. [Source: 나무위키 싸이월드; Cyworld — Wikipedia; ntsc-rs VHS]
- **모티프:** 복원 게이지(today/total 카운터형), 끊긴 BGM 막대, 깨진 사진첩 썸네일, 파도타기 왕복 점프, 도토리(복원 연료), 미니룸 채우기, 일촌평/방명록.

### 3-2. 타이포 방향
- **방향:** 시스템 모노스페이스(카운터·파일명·진행률 = "기계가 복구 중") + 거친 산세리프(다이어리·방명록 본문 = "사람이 쓴 날것")의 혼용. 매끈한 지오메트릭 산세 회피. 싸이 시절 *기본 웹폰트(굴림체 계열)의 정직한 투박함*을 참조 방향으로(특정 폰트 지정 아님 — CEO 자유).
- **가독성 잠금:** 본문 ≥16px. 노이즈/번짐은 *장식 레이어*로만, 읽는 텍스트엔 금지.

### 3-3. 컬러 방향
- **방향:** 한국 후기웹의 *미니홈피 스킨 파스텔 + 미니미/도토리의 아기자기한 톤*을 골격색으로, 그 위에 캠코더 *적목 레드 + 인터레이스 그린/시안*(저화질 비디오 신호색)을 질감으로 얹기. 채도는 "보정 전 직접 플래시"의 거친 평면 대비.
- **금지:** 매끈한 그라데이션·파스텔 *글로우*(Y2K/Aero 표백, docs/04 방향 A 탈락). 강팀 기본 보라→분홍 그라데이션도 미적용(외부 브랜드).
- *주의:* 구체 HEX는 CEO 자유. 위는 *색의 출처*(미니홈피 스킨·비디오 신호색)만 잠금.

### 3-4. 모션 방향
- **방향:** "BGM 버퍼링 채움"·"디인터레이스 복원"·"프로그레시브 디코딩"·"카운터 똑딱"·"파도타기 점프 전환"이 핵심 모션 언어. 매끈한 ease-in-out 슬라이드 ✕.
- **접근성:** `prefers-reduced-motion` 시 지직/스캔라인/점프 애니메이션 정지 → *정적 복원 단계*로 대체. 노이즈 강도 상한(광과민 방지).

### 3-5. ★ 유지 vs 교체 디자인 문법 (하이브리드 갱신)

| 구분 | 유지 (Core 문법 — 다음 era에도 남음) | 교체 (RE:WAVE era 한정 — 다음 era에 갈림) |

### 3-6. ★ 퇴폐 코드 배제 — 시각적 가드레일 (막내 멤버 안전선, rev.1 유지)
리서치로 *분리선* 확정: indie sleaze 시각은 **(A) 촬영 기술 질감**과 **(B) 퇴폐 맥락 코드**로 나뉜다. A만 발굴, B 차단. [Source: Aesthetics Wiki — Indie Sleaze; Wikipedia — Indie sleaze]

| 발굴 OK — 촬영/디지털 *질감* | 명시 배제 — *퇴폐 맥락* 코드 |
|---|---|
| 직접 플래시의 평평한 대비·하드 섀도 | 클럽·파티·군중 씬, 어두운 술집 배경 |
| 적목(red-eye) — *사물/화면*에 적용 | 적목 *난 취한 인물 얼굴* 연출 |
| 저화질 그레인·JPEG 번짐·인터레이스 | 담배·술병·약물·재떨이 소품 |
| 캠코더 타임스탬프·웹캠 셀카(밝고 일상) | 땀·번진 화장·"waif-thin" 신체 미화 |
| 미니홈피·미니룸·BGM·방명록·일상 셀카 | voyeuristic(훔쳐보는) 파티 다큐 시선 |

- **운영 규칙:** "날것 = 정직한 혼돈(디지털 질감)"이지 "퇴폐(인간 행위)"가 아니다. 한국 후기웹 핵심 모티프(미니홈피·미니룸·일상 셀카)는 *밝고 일상적*이라 가드레일과 천연 정합 — 어둠·취기·클럽은 진입 금지. CEO·아뱅 공통 준수.
- **시각 체크 1줄:** "이 컷에 술/담배/클럽 어둠이 있나?" → 있으면 배제. "질감은 거칠되 장면은 일상(미니홈피적)인가?" → 예여야 통과.

#### (3) 혼동 테스트 (모든 컷·컴포넌트 출고 전 자가 적용 — docs/09 §b 규칙3)
> **"일반 소비자가 '이거 싸이월드가 만든/제휴한 거냐?'고 오인할 여지가 있나?"** → 있으면 후퇴.
- 1줄 체크: ① 싸이 고유명사 노출? ② 미니미/도토리 *형상* 닮음? ③ 싸이 특유 배색·로고 닮음? — 하나라도 YES면 추상화 더.
- 통과선: "그 시절 미니홈피 *느낌*"은 살되, "*싸이월드 그 자체*"로 특정되지 않는다. (정서는 명칭이 아니라 구조·BGM·카운터 경험에서 온다 — docs/09 §c)

### 6-2. KiiiKiii 자체 비주얼 — UX 역할 먼저 → 시각 스펙 (3개 자산)

> 흐름(§1)에서 각 자산이 *어느 단계 어느 행동*을 맡는지부터. 비주얼은 그 행동을 돕기 위해서만.

#### 자산 A — **PIXEL SELF (도트 분신)** · 미니미 대체
- **UX 역할:** ③ 공동운영 단계의 *기여 가시화*. "복원 by @핸들"의 얼굴 = 익명 군중이 아닌 *이름 있는 발굴자*(§1-3 이탈③ 대응).
- **시각 스펙:** 16×16 또는 24×24 정사각 도트 그리드, 단순 기하 실루엣. 5인 멤버 컬러 변주는 `--retro-*` 신호색에서. **안전선:** 미니미 2등신 큰머리 비례·시그니처 표정셋·미니룸 가구 세트 *형상 회피* → KiiiKiii 자체 도트 룩(평면 픽셀, 캐릭터 아닌 *분신 토큰*).
- **상태:** `empty`(미설정=회색 깨진 도트 `--state-broken`) / `loading`(도트가 위→아래 프로그레시브로 채워짐) / `restored`(풀컬러).

#### 자산 B — **KEY 아이콘** · 도토리 대체
- **UX 역할:** ② 복원 단계의 *언락 연료*. "KEY 1개 = 깨진 1칸 개봉"(§1-2). RESTORE METER(§2-1) 적립 단위.
- **시각 스펙:** 8비트 도트 **열쇠** 모티프(Lucide `key` 베이스 → 도트화). 색 = `--primary`(앰버골드 = 열쇠 금속 + 도토리 따뜻함을 *색으로만* 계승). 개봉 모션 = 열쇠 돌아가며 칸이 디인터레이스 복원. **안전선:** acorn(도토리) 실루엣 절대 회피 — 열쇠는 KiiiKiii Core 자산이라 *우리 IP에 뿌리*.
- **상태:** `disabled`(KEY 부족=회색 자물쇠) / `loading`(적립 카운터 mono 똑딱) / `active`(앰버 발광 없는 솔리드).

#### 자산 C — **RIDE 아이콘** · 파도타기 대체
- **UX 역할:** ②③의 *왕복 점프* — 다른 발굴자의 복원 중 MY ROOM으로 건너뜀(§2-3). 일방숭배 아닌 티키타카(왕복).
- **시각 스펙:** 물결선 + 건너뛰는 화살표(Lucide `waves` + `arrow-right`/`refresh-cw` 왕복 조합). 색 = `--secondary`(시안 = wave + 비디오 신호). **안전선:** 싸이 "파도타기" 특유 파도 아이콘 형상 회피 → RE:WAVE 물결 = 자체 wave 라인.
- **상태:** `loading`(점프 전환 = 디인터레이스 트랜지션) / `disabled`(비로그인=회색 + "발굴자로 합류해야 RIDE 가능") / `focus`(또렷한 포커스 링, 노이즈가 가리지 않게).

### 6-3. 자체 색 팔레트 — Crowny 토큰 위 후기웹 레트로 (시각 위계)

> 첫째 봐야 할 것 = **CTA(개봉/복원 = KEY 앰버 `--primary`)** → 둘째 = **RIDE 시안 `--secondary`(왕복)** → 셋째 = **복원 진행 카운터(mono)**. 매끈 글로우 그라데이션 금지(§3-3) → era CTA는 *솔리드 하드 플래시 대비*.

| 레이어 | 토큰 | 출처(잠금) |
|---|---|---|
| 골격(스킨판·종이) | `--minihome-skin` `--minihome-paper` `--minihome-line` | 일반 후기웹 파스텔(싸이 특유색 회피) |
| 브랜드 CTA | `--primary`(KEY 앰버) / `--secondary`(RIDE 시안) | KEY=열쇠 자산 / RIDE=RE:WAVE |
| 질감 강조 | `--retro-redeye`(사물·화면만) `--retro-green` `--retro-cyan` | 캠코더 비디오 신호색(§3-3) |
| 노이즈 레이어 | `--retro-scanline` `--retro-static` | 콘텐츠 위 *별도 레이어*(가독성 분리) |
| 깨짐=디폴트 | `--state-broken` `--state-broken-bg` `--state-restoring` `--state-restored` | docs/06 §4-1 깨짐 토큰화 |

### 6-4. 토큰 매핑 — 어디에 무슨 토큰 (강개발 핸드오프, 픽셀 아닌 토큰명)

| 화면 요소 | 토큰 | 비고 |
|---|---|---|
| RESTORE METER 채움(내 기여) | `--state-restored`(=`--primary`) | 즉시 미세 보상 |
| RESTORE METER 채움(전체 누적) | `--secondary` | 팬덤 누적 게이지 |
| 미복원 칸·스켈레톤 | `--state-broken` / `--state-broken-bg` | "깨짐=디폴트" |
| 복원 처리중 하이라이트 | `--state-restoring`(=`--secondary`) | 디인터레이스 |
| today/total 카운터·진행률·파일명 | `--font-family-mono` + `--font-md`(16px) | "기계 복구" 질감, 16px 하한 |
| 다이어리·방명록 본문 | `--font-family-base` Pretendard `--font-md` | 사람 날것, 16px |
| 미니홈피 패널·카드 | `--radius-xl`(16px) + `--minihome-skin` + `--shadow-md` | |
| KEY/RIDE 버튼 | `--radius-md`(10px) | CTA = 솔리드(글로우 금지) |
| 적목 강조(화면·사물) | `--retro-redeye` | *인물 얼굴 금지*(§3-6) |
| 스캔라인/지직 오버레이 | `--retro-scanline` `--retro-static` | 콘텐츠 위 별도 레이어 |
| 비로그인 disabled | `--text-muted` + `not-allowed` | "발굴자로 합류" 카피 |
| 에러(복원 실패) | `--error` `--error-light` | "또 끊겼다" 세계관 카피 |

### 6-5. 토큰 코드화 산출물 — `styles/tokens.css` (레포 루트)

- **위치:** `styles/tokens.css` (프로젝트 로컬 = 스타일 카탈로그 README §40이 가리키는 그 파일).
- **구조:** `[A] BASE`(Crowny Class 시스템 토큰 전체 — 스타일 #1 §15 그대로, 변경 금지) + `[B] THEME`(RE:WAVE era 오버라이드 — §18 방식으로 브랜드 2색만 교체 + `--retro-*`/`--minihome-*`/`--state-*`/모노 추가).

- **박은 토큰 요약:** ① BASE 시스템 토큰 전부(radius 10/16, spacing 8그리드, shadow, font scale 16px 하한, transition, layout, Pretendard/Agbalumo) ② era 브랜드 `--primary`(KEY 앰버 #E5912B)/`--secondary`(RIDE 시안 #1AA6B7) ③ 레트로 질감 `--retro-redeye/green/cyan/scanline/static` ④ 미니홈피 골격 `--minihome-skin/paper/line` ⑤ 깨짐 상태 `--state-broken/-bg/restoring/restored` ⑥ 카운터용 `--font-family-mono`(시스템 스택 — 새 웹폰트 도입 아님).
- **CEO 재량:** era HEX는 docs §3-3 색 출처에 묶인 *방향 제안값* — 픽셀 확정 시 토큰 *이름·구조 유지하고 값만 교체*.
