## 3. 의도적으로 원명칭을 *남긴* 곳과 이유 (= 혼동 유발 아님)

> 원칙: **착안은 인용(원명칭+출처), 브랜딩은 자체 명칭.** 아래는 캠페인 표면이 아니라 *분석·인용·법무 고지*라 docs/09·10과 동일하게 원문을 보존한다. 원명칭을 자체명으로 바꾸면 *허위 사실*이 되거나(예: "KEY가 1곡 6개였다") 리스크 설명이 무너진다.

1. **docs/04 §1-3 "한국 후기웹 메커니즘" 리서치 표** — 실제 서비스의 사실(가격·기능)을 출처와 함께 기록한 *원천 리서치*. 표 상단에 "착안은 인용, 브랜딩은 자체명" 배너 부착.
2. **docs/05 §6 / PROPOSAL §9 R1 — IP 리스크 행 & Plan B 범례** — "어떤 원명칭을 피했는지"를 명시해야 리스크 설명이 성립. 본문 치환 완료를 함께 적시.
3. **PROPOSAL §9 도입부 — 리스크 고지 1문장** — "원천 IP(싸이월드 계열)가 최우선 리스크 → Plan B로 전면 치환" 정직 고지.
4. **각주 [^9][^17][^18] — 출처 인용** — 실제 기사(뉴시스·나무위키·thedrive 등)를 가리키는 citation. URL·출처 성격상 치환 불가, 변경 시 검증 단절.

→ 위 4종은 모두 *카피·해시태그·앨범명·UI·era 앵커가 아님*. **컨셉/캠페인 본문의 혼동 유발 직접 명칭 = 0.**

## 4. 제안 핵심 카피 3줄 (KEY·RIDE 동사 정합)

> 톤: 정직·날것(grime)·공동운영. 손실회피 대신 *획득 프레임*(docs/08 §5-3 윤리 가드 준수).

- **A (메인·공동운영):** *"닫힌 홈피 하나, 너의 물결 한 번에 열린다. KEY는 네 손에."* — RIDE("탄다")+KEY("연다")를 한 줄에. CTA 내장.
- **B (호기심·0단계 누수):** *"누가 이 홈피를 닫았을까. 마지막 방문자가 너라면, 다시 켤 차례."* — 호기심 간극, 싸이월드 비노출로도 "그 시절 홈피" 즉시 환기.
- **C (왕복·관계):** *"네가 물결을 타고 건너올 때마다, 멤버의 RE: 한 줄이 돌아온다."* — 상호성(이웃 한 줄=멤버 답방), TiiiKiii 핑퐁 직격.

> 적용 원칙: #1(통념 깨기 — 트레이드마크 향수에 기대지 않고 *경험·동사*로 anemoia 재점화) / 당긴 심리 레버: 호기심 간극 + 상호성 + 공동 성취(소속감) / 수익 경로: 전환↑(CTA형 카피→복구 클릭→pre-save 환류) · CAC↓(물결타기 오가닉 확산)
