## 0. 왜 색이 서사를 떠받쳐야 하나 (3종 공통 기준)

RE:WAVE 서사 = **복원(decay→restore) · 왕복(RIDE/티키타카) · anemoia(겪지 않은 향수)**. 색의 일은 3가지다.
1. **KEY(`--primary`)** = 열쇠/언락 연료 → *따뜻하고 손이 가는* CTA 색(첫째 시선). 도토리 '명칭/형상'은 회피하되 *따뜻한 색 정서*는 KiiiKiii Core "열쇠" 자산으로 계승.
2. **RIDE(`--secondary`)** = 물결/왕복 점프 → *차갑고 흐르는* 신호색(둘째 시선). KEY와 한난(寒暖) 대비를 이뤄 "개봉(따뜻)↔이동(차가움)"이 색으로 갈림.
3. **broken→restored 상태색** = 회색(깨짐, 디폴트) → KEY/RIDE로 *차오름*. 복원이 곧 채도의 회복.

**싸이 회피 가드(3종 전부 통과):** 싸이 로고 시그니처 = *선명한 주황-빨강(vivid orange-red)*. 3종 KEY는 모두 *그보다 노랗거나(허니/애시드/매리골드) 탁한* 쪽으로 빼서 주황-빨강 직격을 피한다. RIDE는 셋 다 시안/블루 계열 = 싸이 정체성과 무관. → 혼동 테스트 ①②③ 통과(고유명사·미니미/도토리 형상·특유배색 비복제).

- **⚠ 가드 주의:** 파스텔은 §3-3 "파스텔 *글로우* 금지"와 *아슬한 경계* — **반드시 무광 평면·하드 대비**로만 운용(Aero/Y2K 글로시 베벨 절대 금지). 깨짐 상태가 파스텔에 묻히지 않게 `--state-broken` 회색 대비를 또렷이.
- **BASE 충돌 체크:** radius·shadow·폰트 그대로. 파스텔 면 위 16px 본문 대비 = `--text-dark`로 확보(파스텔 위 연한 회색 텍스트 금지, AA 주의). 충돌 없음.
- **다크/라이트 주의:** 파스텔은 *다크 배경에서 탁하고 유아틱* → 다크 모드 시 골격을 딥플럼(`#26222E` 계열)으로 치환하고 KEY/RIDE *채도만* 살림. 라이트가 본령.
